## 🗺️ خريطة التعلم

### للمبتدئين

1. ابدأ بـ [دليل البدء السريع](QUICK_START.md)
2. اقرأ [دليل الإعداد الشامل](KIRO_SETUP.md)
3. استكشف [دليل أدوات MCP](MCP_TOOLS_GUIDE.md)

### للمستخدمين المتقدمين

1. راجع [دليل التكوين](CONFIG_GUIDE.md)
2. اطلع على [دليل أدوات MCP](MCP_TOOLS_GUIDE.md)
3. احتفظ بـ [دليل استكشاف الأخطاء](TROUBLESHOOTING.md) كمرجع

### للمسؤولين

1. اقرأ [دليل التكوين](CONFIG_GUIDE.md) بالكامل
2. راجع قسم الأمان في [دليل الإعداد](KIRO_SETUP.md)
3. تعرف على [دليل استكشاف الأخطاء](TROUBLESHOOTING.md)

## 📋 قائمة التحقق السريعة

قبل البدء، تأكد من:

- [ ] Node.js 18 أو أحدث مثبت
- [ ] Kiro IDE مثبت ويعمل
- [ ] لديك حساب Cirvoy مع صلاحيات API
- [ ] حصلت على API Token من Cirvoy
- [ ] حصلت على Webhook Secret من Cirvoy

## 🆘 الحصول على المساعدة

### إذا واجهت مشكلة:

1. **ابحث في الوثائق**: استخدم البحث في الملفات
2. **راجع استكشاف الأخطاء**: [TROUBLESHOOTING.md](TROUBLESHOOTING.md)
3. **تحقق من السجلات**: `~/.kiro/cirvoy-mcp/logs/server.log`
4. **ابحث في GitHub Issues**: قد تكون المشكلة معروفة
5. **أنشئ Issue جديد**: مع تفاصيل المشكلة

### عند إنشاء Issue:

قم بتضمين:
- وصف واضح للمشكلة
- خطوات إعادة الإنتاج
- رسائل الخطأ من السجلات
- معلومات النظام (OS, Node.js version)
- ملف التكوين (احذف البيانات الحساسة!)
